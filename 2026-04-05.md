# Navi Server Analys — 2026-04-05

Analys av navi-server projektet (version 7) för att identifiera buggar, förbättringsområden och teknisk skuld.

---

## 1. Läckande minne i backgroundSyncPushTimestamps

**Fil:** `server.js` (rad ~280)

**Problem:** Map `backgroundSyncPushTimestamps` växer obegränsat om cleanup-intervallet misslyckas eller om det finns race conditions.

```javascript
const backgroundSyncPushTimestamps = new Map();
// Cleanup var 30:e minut
setInterval(() => {
  // ...
}, 30 * 60 * 1000);
```

**Risk:** Om servern körs länge utan att cleanup kör (t.ex. vid hög last), kan minnet växa okontrollerat.

**Förslag:** Lägg till en guard som kör cleanup oftare eller använd WeakMap där möjligt.

---

## 2. Saknar rate limiting för GitHub API-anrop

**Fil:** `github-webhook.js` (14KB, många API-anrop)

**Problem:** Ingen exponential backoff eller rate limit-hantering vid GitHub API-anrop. Om GitHub returnerar 403/429 kan servern fortsätta göra anrop och bli temporärt blockerad.

**Risk:** GitHub kan blockera IP:n vid för många anrop.

**Förslag:** Implementera rate limiting med exponential backoff i github-webhook.js:
```javascript
const RATE_LIMIT_DELAY = 5000; // 5s bas
const backoff = Math.min(RATE_LIMIT_DELAY * Math.pow(2, retries), 60000);
```

## 3. LLM-coordinator saknar exponential backoff

**Fil:** `llm-coordinator.js`

**Problem:** Om API:et är överbelastat (429), väntar alla requests bara 1 minut (token återställs) men det finns ingen exponential backoff för individuella misslyckade anrop.

```javascript
const timer = setInterval(() => {
  this.tokens = this.requestsPerMinute;
  this._drain();
}, 60_000);
```

**Risk:** Om API:et har problem kan alla requests misslyckas upprepat.

**Förslag:** Lägg till exponential backoff för misslyckade LLM-anrop:
```javascript
if (error.message?.includes('429')) {
  await new Promise(r => setTimeout(r, baseDelay * Math.pow(2, retryCount)));
}
```

## 4. Worker watchdog timeout kan vara för kort

**Fil:** `worker-pool.js` (rad ~10)

```javascript
const WATCHDOG_TIMEOUT = 90 * 60 * 1000;  // 90 min
```

**Problem:** 90 minuter kan vara för kort för långa forskningsjobb eller komplexa byggen.

**Risk:** Långa jobb kan avbrytas i förtid av watchdog.

**Förslag:** Gör timeout konfigurerbar per jobb-typ eller öka till 2 timmar som standard.

---

## 5. SQLite WAL-läge utan regelbunden checkpoint

**Fil:** `storage.js` (rad ~50)

```javascript
this.db.exec(`
  PRAGMA journal_mode = WAL;
  PRAGMA synchronous = NORMAL;
  // ...
`);
```

**Problem:** WAL-läge är bra för concurrency men om servern kraschar kan WAL-filen växa stort. Ingen explicit checkpoint-schemaläggning.

**Risk:** WAL-filen kan bli stor och vid krasch kan dataförlust ske.

**Förslag:** Lägg till periodisk checkpoint:
```javascript
setInterval(() => {
  this.db.exec('PRAGMA wal_checkpoint(TRUNCATE)');
}, 60 * 60 * 1000); // Varje timme
```

## 6. Saknas verktyg för iOS-utveckling

**Fil:** `tools/index.js`

**Problem:** Inga verktyg för att köra Xcode-kommandon, hantera CocoaPods, eller bygga iOS-projekt.

**Risk:** Workers kan inte bygga eller testa iOS-kod.

**Förslag:** Lägg till verktyg:
- `xcodebuild_build` — Bygg iOS-projekt
- `pod_install` — Kör CocoaPods
- `xcodebuild_test` — Kör tester

## 7. Push-service saknar tydlig felindikation vid felkonfiguration

**Fil:** `push-service.js`

**Problem:** Om APNS-nyckeln är fel eller saknas, loggas bara en varning men ingen tydlig indikation till användaren.

```javascript
if (!keyPath || !fs.existsSync(keyPath)) {
  this.logger.warn('apns-disabled-missing-key');
  return;
}
```

**Risk:** Ted kanske inte märker att push-notiser inte fungerar.

**Förslag:** Lägg till en health check för push-service som visar status tydligt.

---

## 8. Worker MAX_ITERATIONS kan vara för lågt

**Fil:** `worker.js` (rad ~20)

```javascript
const MAX_ITERATIONS = 200;
```

**Problem:** 200 iterationer kan vara för lite för komplexa jobb som kräver många steg.

**Risk:** Jobb avbryts i förtid med "Nådde max iterationer".

**Förslag:** Gör MAX_ITERATIONS konfigurerbar eller öka till 500.
