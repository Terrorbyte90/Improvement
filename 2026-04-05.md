# Navi-Server Analys: Buggar och Förbättringar
**Datum:** 2026-04-05

---

## Sammanfattning
Analys av navi-v7 server (version 2.1.0) på Ubuntu med Node.js 20+. Projektet innehåller ~48000 rader kod fördelat på server.js, orchestrator.js, worker.js, worker-pool.js, storage.js, memory.js, push-service.js och verktyg.

---

## 🔴 BUGGAR (HÖG PRIORITET)

### 1. APNS-nyckel saknas — Push fungerar inte
**Allvarlighet:** Kritisk  
**Fil:** `.env` + `push-service.js`  
**Problem:** APNS-nyckelfilen (`/root/navi-v7/AuthKey_apns.p8`) som definieras i `APN_KEY_PATH` finns inte på servern. Detta gör att push-notifikationer inte fungerar alls.

**Bevis:**
```javascript
// push-service.js rad 14-18
const keyPath = process.env.APN_KEY_PATH;
if (!keyPath || !fs.existsSync(keyPath)) {
  this.logger.warn('apns-disabled-missing-key');
  return;
}
```

**Logg:**
```
{"ts":"2026-04-04T23:42:15.870Z","level":"info","component":"push-service","message":"background-push-skipped-no-provider-or-token","reason":"job_progress"}
```

**Åtgärd:** Ladda upp APNS-nyckelfilen till servern eller skapa en ny via Apple Developer Portal.

---

### 2. SQLite Experimental Warning i loggar
**Allvarlighet:** Låg  
**Fil:** `storage.js`  
**Problem:** Node.js 20+ visar varningar om att SQLite är en experimentell funktion.

**Bevis:**
```
(node:733683) ExperimentalWarning: SQLite is an experimental feature and might change at any time
```

**Åtgärd:** Ignorera om det fungerar, eller överväg att byta till annan databas när stabilitet krävs.

---

### 3. Memory Leak: backgroundSyncPushTimestamps
**Allvarlighet:** Medel  
**Fil:** `server.js` rad 268-275  
**Problem:** Mapen `backgroundSyncPushTimestamps` växer obundet om den inte städas. Det finns en prune-funktion men den körs bara var 30:e minut.

```javascript
// server.js rad 268-275
const syncCutoff = Date.now() - 2 * 60 * 60 * 1000; // 2 hours
for (const [key, ts] of backgroundSyncPushTimestamps) {
  if (ts < syncCutoff) backgroundSyncPushTimestamps.delete(key);
}
```

**Åtgärd:** Sänk intervallet till 15 minuter eller minska cutoff till 1 timme.

---

### 4. Möjlig Race Condition i Worker Queue
**Allvarlighet:** Medel  
**Fil:** `worker-pool.js`  
**Problem:** När en worker avslutas och nästa startas från kön kan det uppstå race conditions om flera workers avslutas samtidigt.

```javascript
// worker-pool.js rad 130-140
_dequeueNext() {
  if (this.queue.length > 0 && this.activeCount() < MAX_WORKERS) {
    const next = this.queue.shift();
    this._startWorker(next);
    // Uppdatering av kö-positioner sker asynkront
  }
}
```

**Åtgärd:** Lägg till mutex eller synkronisera åtkomst till kön.

## 🟡 FÖRBÄTTRINGAR (MEDEL PRIORITET)

### 5. Saknas Health Check för Worker Pool
**Fil:** `server.js`  
**Problem:** `/health` endpoint returnerar inte information om worker poolens hälsa.

**Åtgärd:** Lägg till worker pool-metrics till health endpoint.

---

### 6. Saknas Rate Limiting för Orchestrator
**Fil:** `orchestrator.js`  
**Problem:** Ingen begränsning av antal förfrågningar per session, vilket kan leda till API-utarmning.

```javascript
// Finns visserligen sessionQueueDepths men max 5 per session är lågt
const pendingDepth = this.sessionQueueDepths.get(sessionId) || 0;
if (pendingDepth >= 5) {
  const busyMessage = 'Navi är för upptagen just nu...';
}
```

**Åtgärd:** Implementera mer sofistikerad rate limiting med expontiell backoff.

---

### 7. Saknas Automatisk Backup av SQLite
**Fil:** `storage.js`  
**Problem:** Ingen automatisk backup av databasen. Vid server-krash kan data gå förlorad.

**Åtgärd:** Lägg till daglig backup till `/root/navi-v7/backups/`.

---

### 8. Saknas Felhantering för GitHub Webhook
**Fil:** `github-webhook.js`  
**Problem:** Om webhook-hanteraren kastar ett fel kan hela servern påverkas.

**Åtgärd:** Wrappa webhook-hanteringen i try/catch med proper felhantering.

---

### 9. Saknas Logging av API-kostnader
**Fil:** `orchestrator.js`, `worker.js`  
**Problem:** Ingen tracking av API-anrop och kostnader till OpenRouter.

**Åtgärd:** Lägg till metrics för token-användning och kostnadsuppföljning.

---

### 10. Saknas Timeout för vissa LLM-anrop
**Fil:** `orchestrator.js`  
**Problem:** Vissa API-anrop har timeout på 15s men det kan vara för kort för komplexa förfrågningar.

```javascript
// orchestrator.js rad 130
req.setTimeout(15000, () => { req.destroy(new Error('Orchestrator request timeout')); });
```

**Åtgärd:** Öka timeout till 30s för komplexa förfrågningar.

---

### 11. Saknas Caching av Knowledge Context
**Fil:** `knowledge-loader.js`  
**Problem:** Kunskapskontexten byggs om för varje förfrågning utan caching.

**Åtgärd:** Implementera Redis eller minnesbaserad cache för knowledge context.

---

### 12. Saknas Felhantering för Minnesallokering
**Fil:** `worker.js`  
**Problem:** Ingen kontroll av minnesanvändning innan stora operationer.

```javascript
// Lägg till före minnesintensiva operationer
const memUsage = process.memoryUsage();
if (memUsage.heapUsed > 1.5 * 1024 * 1024 * 1024) { // 1.5GB
  this.logger.warn('memory-pressure', { heapUsed: memUsage.heapUsed });
}
```

**Åtgärd:** Lägg till minnesövervakning och graceful degradation.

---

### 13. Saknas Retry-logik för Database Connections
**Fil:** `storage.js`  
**Problem:** Om SQLite-anslutningen går förlorad finns ingen automatisk återanslutning.

**Åtgärd:** Lägg till retry-logik med exponential backoff för databasoperationer.

---

### 14. Saknas Graceful Shutdown för Workers
**Fil:** `worker-pool.js`  
**Problem:** Vid server-shutdown avslutas inte workers korrekt, de får bara SIGTERM.

```javascript
// Finns visserligen gracefulShutdown i server.js men den är begränsad
async function gracefulShutdown(signal) {
  // ...
  // Don't kill them — they run independently and will finish
}
```

**Åtgärd:** Ge workers möjlighet att spara sitt tillstånd innan avslut.

---

### 15. Saknas Metrics för LLM Coordinator
**Fil:** `llm-coordinator.js`  
**Problem:** Det finns ingen detaljerad metrics för LLM-anrop (antal, kostnad, latency).

**Åtgärd:** Lägg till detailed metrics till `/metrics` endpoint.
